# XERP 2026-07-15 Demo Runbook

## 1. 演示目标

在 5 分钟内从固定数据重置开始，完成“报进展 → 提案确认 → 问客户 → 发材料 → 提案确认 → 经营概览 → 权限拒绝”，证明正式事实只在人工确认后写入，回答可追溯，并且全程不依赖飞书、OpenAI 或外网。

## 2. 演示前检查

在 `agent-backend/` 执行：

```bash
./scripts/bootstrap.sh
uv run ruff check .
uv run pytest
uv run python scripts/smoke_demo.py
./scripts/run_demo.sh
```

看到以下输出后保持终端运行：

```text
启动 XERP Demo：http://127.0.0.1:8000/demo
```

浏览器打开 `http://127.0.0.1:8000/demo`。`GET /health` 应包含：

```json
{"status":"ok","service":"xerp-demo","mode":"offline-demo","extractor_provider":"demo"}
```

## 3. 现场操作顺序

### Step 1：重置并确认初始状态

1. 身份选择 `u-demo-owner · 已授权`。
2. 点击“重置 Demo 数据”。
3. 预期：活跃商机、金额线索、待办行动、待确认提案均为 0；仅保留固定风险和 `demo_seeded` 审计。

### Step 2：报进展并确认写入

1. 保持默认文本：`今天拜访了李总。客户希望财务和采购优先，预算约一亿元，月底提交整体方案。`
2. 点击“生成商务进展提案”。
3. 讲解：提案列出关键人、商机、预算线索、行动承诺、缺失字段；此时已确认事实仍为空，待确认提案为 1。
4. 点击“确认并写入”。
5. 预期：活跃商机 1、金额线索 ¥1.0 亿、待办行动 1、待确认提案 0；事实区出现李总、商机和月底方案行动；审计出现 `proposal_confirmed`。

### Step 3：问客户

1. 保持默认问题：`这个客户目前最关键的三个突破口是什么？`
2. 点击“生成有证据的回答”。
3. 逐项指出：数据截止时间、事实/单一来源、Agent 推断、建议动作、来源和缺失信息。
4. 强调预算仍是 `single_source`，没有被表述为客户正式确认事实。

### Step 4：发材料并确认写入

1. 保持默认文件名 `集团数字化规划.pdf` 和默认说明。
2. 点击“生成资料归档提案”。
3. 讲解：此时资料证据仍为 0，待确认提案为 1。
4. 点击“确认并写入”。
5. 预期：资料证据变为 1、状态 `archived`，待确认提案归零，并新增确认审计。

### Step 5：经营概览

确认页面展示：

- 活跃商机：1
- 金额线索：¥1.0 亿
- 待办行动：1，逾期 0
- 高风险：0（固定 medium 风险仍在事实区）
- 待确认提案：0
- 经营摘要包含中国星海集团、商机金额和行动数量

### Step 6：权限拒绝

1. 将身份切换为 `u-outsider · 未授权`。
2. 预期：页面提示“无权访问该客户”。
3. KPI 全部变为 `—`；提案/回答、事实、经营摘要和审计不再显示原授权数据。
4. 可说明浏览器网络面板会看到预期的 dashboard/snapshot 403，但不应有 JavaScript 异常。

### Step 7：收尾复位

1. 切回 `u-demo-owner · 已授权`。
2. 点击“重置 Demo 数据”。
3. 预期回到 Step 1 初始状态，便于下一场演示。

## 4. 故障回退

| 现象 | 处理 |
|---|---|
| 8000 端口占用 | `PORT=8010 ./scripts/run_demo.sh`，打开对应 `/demo` |
| 页面状态不确定 | 先切回授权用户，再点击“重置 Demo 数据” |
| 服务未启动 | 重新执行 `./scripts/run_demo.sh`，检查终端错误 |
| 依赖缺失 | 重新执行 `./scripts/bootstrap.sh` |
| 外网、飞书或模型不可用 | 无需处理；P0 固定使用 SQLite、Fake Feishu Replay 和 DemoExtractor |
| 现场数据被改乱 | 点击 reset；如仍异常，停止服务后删除 `data/demo.db` 再启动（仅限本 Demo 数据库） |

## 5. 彩排记录要求

每次彩排必须：

1. 从 reset 开始。
2. 完整执行 Step 2—6。
3. 检查未授权身份不会看到先前加载数据。
4. 最后再次 reset。
5. 在 `.planning/demo-2026-07-15/progress.md` 记录时间、结果和任何偏差。
