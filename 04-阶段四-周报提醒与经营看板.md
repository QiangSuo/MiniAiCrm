# 阶段四：周报提醒与经营看板

## 1. 阶段目标

在三个核心闭环稳定后，增加轻量经营自动化能力，让系统从“资料和进展记录工具”升级为“客户经营辅助工具”。

本阶段重点：

- 自动客户周报。
- 逾期行动提醒。
- 风险提醒。
- 商机健康度检查。
- 关键人覆盖度分析。
- Base 视图和轻量看板。

## 2. 建议周期

4-7 天。

本阶段优先做规则型自动化，不做复杂预测。Codex 负责实现扫描、汇总、卡片和视图说明；你负责确认提醒频率、周报格式和管理者看板口径。

## 3. 自动客户周报

### 周报来源

从以下表读取：

```text
接触记录
资料证据
商机
行动承诺
风险
关键人
变更提案
```

### 周报内容

```text
本周新增资料
本周新增接触记录
关键人变化
商机阶段变化
新增或升级风险
逾期行动和承诺
下周重点行动
需要管理者协调事项
Agent 判断和理由
```

### 流程

```text
定时任务触发
        ↓
按客户读取本周变化
        ↓
生成周报草稿
        ↓
发送给客户负责人确认
        ↓
确认后推送客户作战群
        ↓
可选归档到 Wiki 分析报告目录
```

### 周报确认卡

```text
中国XX集团客户周报

本周新增进展：3 条
新增资料：2 份
逾期行动：1 条
新增风险：1 条
需协调事项：2 条

[发送到作战群] [修改] [仅保存] [取消]
```

### 验收标准

- 周报不是流水账，必须突出变化、风险、行动。
- 周报中每个关键结论能追溯到 Base 记录。
- 客户负责人确认后再推送。

## 4. 逾期行动提醒

### 扫描规则

每日或每周扫描 `行动承诺` 表：

```text
截止日期 < 今天
且状态不等于 已完成 / 取消
```

### 提醒对象

- 事项负责人。
- 客户负责人。
- 可选：客户作战群。

### 提醒卡内容

```text
以下行动已逾期：

客户：中国XX集团
事项：月底前提交整体方案
负责人：张三
截止日期：2026-07-31
逾期天数：3 天
关联商机：财务共享项目

[标记完成] [申请延期] [补充说明] [关闭]
```

### 验收标准

- 逾期行动不会遗漏。
- 提醒对象准确。
- 用户可以通过卡片更新状态。
- 更新动作写回 `行动承诺` 表。

## 5. 风险提醒

### 风险来源

- 用户报进展时生成的风险提案。
- 商机长期无进展。
- 行动逾期。
- 关键人关系恶化。
- 预算、招标、竞争、合规等资料变化。

### 风险等级

```text
高
中
低
```

### 风险提醒规则

第一版先做规则，不做复杂预测：

```text
高风险新增 -> 立即提醒
中风险新增 -> 周报汇总
风险超过 14 天未更新 -> 提醒责任人
高金额商机无下一步行动 -> 风险提示
```

### 验收标准

- 风险必须有来源或触发依据。
- 高风险必须有责任人和应对动作。
- 风险提醒不能泛滥，避免变成噪音。

## 6. 商机健康度检查

### 检查项

针对 `商机` 表：

```text
是否有明确需求
是否有预算线索
是否有关键人支持
是否有采购路径
是否有下一步行动
是否有竞争信息
是否有证据来源
最近 30 天是否有接触记录
```

### 输出

可以生成：

```text
健康
需补证据
阶段可能高估
推进停滞
高风险
```

### 对应动作

- 无预算线索：建议补充预算验证行动。
- 无关键人：建议补充关键人覆盖。
- 无下一步行动：建议创建行动承诺。
- 30 天无接触：建议安排客户沟通。

### 验收标准

- 商机健康度不是黑盒，必须展示检查项。
- 建议动作可以生成变更提案或行动承诺。

## 7. 关键人覆盖度分析

### 指标

从 `关键人` 表和 `接触记录` 表计算：

```text
已识别关键人数
高影响力关键人数
已接触关键人数
支持 / 中立 / 反对人数
超过 60 天未接触人数
财务、采购、信息化等关键角色覆盖情况
```

### 输出

```text
关系覆盖完整
缺财务关键人
缺采购关键人
高影响力关键人未接触
关系存在风险
```

### 验收标准

- 能指出缺哪些角色。
- 能给出下一步拜访建议。
- 关系判断必须标注来源或置信度。

## 8. Base 视图和轻量看板

第一版尽量使用 Base 视图和仪表盘，不建设独立管理后台。

### 推荐视图

客户表：

```text
我的客户
重点客户
谨慎保留客户
无作战群客户
无 Wiki 客户
```

商机表：

```text
活跃商机
高金额商机
阶段可能高估
无预算证据
30天无进展
```

行动承诺表：

```text
我的待办
本周到期
已逾期
无负责人
高优先级
```

风险表：

```text
高风险
新增风险
处理中风险
无应对动作
```

变更提案表：

```text
待我确认
有冲突提案
写入失败
本周已确认
```

### 管理看板

第一版看板可展示：

```text
客户总数
活跃客户数
活跃商机数
预计商机金额
逾期行动数
高风险客户数
待确认提案数
本周新增进展数
```

### 验收标准

- 客户负责人能从视图看到自己的待办和客户状态。
- 部门管理者能看到重点客户、风险和商机概览。
- 不需要独立后台也能完成日常运营。

## 9. 阶段验收标准

- 至少 3 个客户可生成周报。
- 逾期行动提醒可用。
- 高风险提醒可用。
- 商机健康度检查可用。
- 关键人覆盖度分析可用。
- Base 视图能支撑客户负责人和管理者日常查看。

## 10. Codex 开发任务拆分

| 顺序 | 任务 | 验收方式 |
| --- | --- | --- |
| 1 | 逾期行动扫描 | 能列出本周到期和已逾期事项 |
| 2 | 逾期提醒卡片 | 能通过卡片更新状态 |
| 3 | 高风险扫描 | 高风险新增或长期未更新时提醒 |
| 4 | 客户周报生成 | 生成变化、风险、行动和协调事项 |
| 5 | 周报确认卡 | 客户负责人确认后推送 |
| 6 | 商机健康度检查 | 输出检查项和补证据建议 |
| 7 | 关键人覆盖度检查 | 输出缺口和下一步拜访建议 |
| 8 | Base 视图整理 | 形成客户负责人和管理者视图 |

## 11. 阶段风险

| 风险 | 应对 |
| --- | --- |
| 周报太长没人看 | 周报只突出变化、风险、行动和协调事项 |
| 提醒过多 | 设定提醒等级和频率 |
| 健康度评分被误解 | 定义为证据完整度，不直接否定业务判断 |
| Base 看板能力不够 | 第一版做固定视图，复杂 BI 后置 |
| 自动化逻辑分散 | 规则集中放后端，不大量依赖 Base workflow |
